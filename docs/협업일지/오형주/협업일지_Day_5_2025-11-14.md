## [9] 추가 기록 (선택사항)
**답변**: 
```
streamlit-extras: Streamlit에 없는 작은 기능 및 유틸리티 추가
장점: 간편, 다양, 통합
단점: 제한된 범위(UI 컴포넌트 폭이 작음), 개별 기능 (하나의 거대한 테마가 아닌 개별 기능 위주)

streamlit-antd-components: Ant Design 기반의 풍부하고 정교한 컴포넌트 추가
장점: 고급스러움, 다양한 컴포넌트, 라이트/다크 모드 지원
단점: 복잡한 학습 곡선, Ant Design에 상당히 의존적

st-pages: 멀티 페이지 앱의 유연한 관리 및 네비게이션 커스텀
장점: 페이지 관리 용이, 다양한 유틸리티, 기존 streamlit 앱 기능보다 유연
단점: 제한된 범위(멀티 페이지 관리 및 네비게이션 기능 위주)

styles.css: 순수한 시각적 스타일(색상, 레이아웃, 애니메이션 등)을 정의합니다. 디자이너/디자인 변경 시 이 파일만 수정하면 됨.

streamlit_styling.py: Streamlit 환경에서 CSS를 주입하고, 공통 UI 조각(메트릭 카드, 알림, 메시지 버블 등)을 HTML로 생성해 재사용 가능한 함수로 제공하는 헬퍼 라이브러리입니다. 로직/템플릿/안전 검사(파일 존재 확인 등)를 담당합니다.
```
