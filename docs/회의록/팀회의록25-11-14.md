## 주요 안건 및 논의사항

### 1. 신승목 진행상황
- **HWP → PDF 변환 완료**
  - 변환된 파일을 팀원들과 공유 예정

### 2. 김명환 진행상황 (상세)

#### 2.1 아키텍처 업데이트
- **데이터베이스 단순화**
  - PDF/마크다운 저장 DB
  - 채팅 DB
  - 벡터 DB (임베딩용)
- **벡터 데이터 매니저 개발**
  - 해시 기반 청크 관리
  - 중복 임베딩 방지 기능

#### 2.2 임베딩 및 전처리
- **유사도 점수 정규화 검토**
  - 정규화 시 수치가 너무 낮아짐 (0~1)
  - 거리 기반(0~무한대) 방식이 더 효과적으로 판단
  - 최종: 거리 기반 방식 유지

- **마크다운 전처리 함수 개발**
  - 테스트 블록, 코드 블록 처리
  - 불필요한 요소 제거
  - 유닛테스트 완료

#### 2.3 임베딩 프로세스 설계
- **청킹 전략 고민**
  - 초안: 페이지 단위 청킹
  - 문제점: 코드 블록, 표 등이 페이지 간 분리될 수 있음
  - 수정안: 마크다운 의미 단위로 분리
  - 임시: 파일 단위로 먼저 테스트 후 의미 단위로 개선 예정

#### 2.4 문서 관리 API
- 문서 이름 + 해시값 기반 정보 조회 API 추가
- 더미데이터 생성 작업 진행 중 (오늘 완료 예정)

#### 2.5 Config 설정 (config.py)
- **중요**: 전체 공통 설정 파일
- 개인별 개발 환경에 맞게 설정 필요
- 프로젝트 루트, 임베딩 위치 등 설정
- OpenAI API 키는 환경변수로 관리 (보안)

#### 2.6 금일 작업 계획
- 전체 파이프라인 검증
- 검색 엔진 개발 (키워드 검색)
- UI 연동 준비
  - 파일 선택/추가 기능
  - DB 및 임베딩 동기화 API
  - 채팅 기능 개발

### 3. 기술 세션: 개발 환경 세팅

#### 3.1 Config 사용법 상세 설명
- `config.py` 구조 및 사용법
- 환경변수 설정 방법
- OpenAI API 키 관리 (환경변수 등록 필수)
- 개인 개발 환경 설정 가이드

#### 3.2 GPT 모델 관련 주의사항
- GPT-4.5-mini로 변경됨 (기존 GPT-4o)
- 토큰 사이즈 설정 중요
  - 100 토큰: 응답 없음 (오류)
  - 충분한 토큰 사이즈 설정 필요
- 모델별 파라미터 테스트 필요

#### 3.3 테스트 방법론
- 개발자는 직접 만든 기능을 충분히 테스트해야 함
- AI를 활용한 테스트 케이스 생성 권장
- 유닛테스트 작성 및 실행 예시 공유

#### 3.4 개발 도구 권장사항
**GitHub Copilot (필수)**
- 유료 가입 권장 (개발 생산성 향상)
- VSCode 연동 방법
- 긴 문맥의 코드 작업: GPT-4.0 또는 고급 모델 사용
- 짧은 함수/디버깅: GPT-4.0으로 충분

**Claude (선택)**
- 문서 작업용으로 활용
- Pro 버전으로도 충분

**IP 체계 기본 지식**
- 개발자 필수 지식으로 강조
- 사설 IP vs 공인 IP
- 포트 포워딩 개념

#### 3.5 VM 환경 설정
- 삼바(Samba) 설정 시도 중
- VM 폴더를 로컬 드라이브처럼 연결 테스트
- 사설 IP (10.x.x.x) vs 공인 IP 이슈
- 대안: FTP 서버 구축 검토

### 4. 체크리스트 관리
- **중요**: 각자 맡은 분야의 체크리스트 확인 필수
- 프로젝트 전체 체크리스트 업데이트 예정 (회의 후)
- PM 역할 분담 (신승목)

### 5. 개발 규칙
- GitHub Copilot 규칙 파일 추가
- 변수 타입, 코딩 스타일 가이드
- 일관된 코드 품질 유지
