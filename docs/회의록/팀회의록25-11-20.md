## 회의 목적

파일 처리 파이프라인(PDF/HWP) 구현, 데이터포털 연동 테스트, 채팅 히스토리 DB 구조/로직 보완, UI 연동 계획 및 문서화/발표 전략을 정리하고 금일/단기 액션 아이템을 확정한다.

---

## 주요 안건 및 논의사항

### 1. 진행 현황 공유
**각자 진행 상황**:
* **신승목**: 어제 작업물 확인 완료. 오늘 PDF/HWP 분기 처리 로직 구현 예정(HWP 모듈 연동해 DB 저장). 데이터포털(데이터 지원 KR) API 키 수령 완료, 샘플 스크립트 기반 테스트 예정.
* **오형주**: 제공된 파이프라인 동작 확인 중. 채팅 히스토리 DB 저장 로직 시도(최신 5개 유지) 중이나 현재 중복 값 저장 문제 발생; 최신순 정렬/저장 로직 보완 필요. 이후 민규님 인터페이스 추가분 평가 예정.
* **김명환**: 데이터포털 수집 샘플 스크립트 안내 및 UI 연동 아이디어 제안(리프레시 버튼, 해시 기반 중복 스킵). HWP → Markdown 변환 도구 서버 배포/테스트 PyPI 경로 공유. 전처리 보고서 작성 완료, 임베딩 문서 정리 계획. 발표 자료는 단일 문서로 통합 제안.

### 2. 핵심 논의사항
**주요 이슈**:
* 파일 업로드 및 처리 로직: PDF는 기존 방식, HWP는 별도 모듈 연동 필요.
* 데이터포털 연동: 기간 기반 수집, UI에서 리프레시 버튼으로 신규만 추가(중복 처리).
* 채팅 히스토리 DB: 최신 5개 저장 로직에서 중복 저장/정렬 이슈.
* 문서화/발표: 산출물 문서 통합 전략.

---

## 주요 결정사항

* UI에 데이터포털 연동 리프레시 버튼을 추가하고, 해시 기반 중복 제거 방식 적용.
* 채팅 히스토리 저장은 최신순 5개 유지 원칙으로 정렬/제한 로직 확정, 타임스탬프 필드 활용 (**ORDER BY updated_at DESC LIMIT 5**). 필요 시 김명환이 DB 설계/쿼리 리뷰 지원.
* 발표 자료는 단일 문서로 통합 관리.
* HWP 처리 모듈을 파이프라인에 연동하여 DB 저장까지 완료하는 방향으로 개발 진행.
* 데이터포털 수집은 제공된 샘플 스크립트로 우선 검증 후 서비스 연동.
