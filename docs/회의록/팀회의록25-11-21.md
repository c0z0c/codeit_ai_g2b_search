## 주요 안건 및 논의사항

### 1. 진행 현황 공유
**각자 진행 상황**:
- 김명환: 발표 자료 쪽 전처리 보고서 제작, 임베딩 설계서 작성, HWP 변환 툴 (마크다운, 텍스트) 제작 및 공유
- 신승목: 원본 데이터(hwp 96개 + pdf 4개) 마크다운 변환 후 DB 저장 구조 변경 중(기존 PDF에서 HWP도 처리하면서 페이지 정보를 넣을 수 있도록 추가)
- 오형주: DB쪽 확인, RAG와 연결되는 UI를 만들어서 테스트
- 이민규: 파이프라인 검수(채팅과 추가 질의가 계속 연동되도록 점검)

### 2. 핵심 논의사항
**주요 이슈**:
HWP, PDF 통합 마크다운 변환 후 DB 저장 과정을 재구성하다가 PDF가 저장되지 않는 문제 발생
**논의된 해결방안**:
이전에 PDF가 처리되던 구조에서 HWP 처리 과정을 추가
HWP는 페이지 정보가 없기 때문에 80줄마다 page_number 들어가야 함
**팀 결정사항**:
발표와 보고서용 PPT 제작, 현재 프로젝트 정도면 2-3일 정도에 가능
발표 자료에 UI쪽 내용을 캡쳐해서 추가
발표 자료에 애니메이션 어느 정도 추가
