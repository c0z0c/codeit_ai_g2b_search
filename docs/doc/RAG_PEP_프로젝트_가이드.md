## 1. 프로젝트 개요

### 1.1. 프로젝트 목표

RAG 시스템을 구축하여 PEP 문서를 수집, 전처리, 임베딩하고, 사용자 질의에 정확한 답변 제공

### 1.2. 프로젝트 기간

- **기간**: 3주 (2025-11-10 ~ 2025-11-28)
- **Week 1**: 프로젝트 기반 구축, DB 스키마, 더미 데이터, UI 프로토타입
- **Week 2**: 핵심 기능 개발 (문서 처리, 임베딩, LLM 통합)
- **Week 3**: 통합 테스트, 성능 최적화, 문서화, 발표 준비

---

## 2. 시스템 아키텍처

### 2.1. 팀 역할 분담

| 역할          | 담당자       | 핵심 업무                                      |
|---------------|--------------|-----------------------------------------------|
| 데이터 엔지니어 | 신승목 | 문서 수집 및 원본 전처리 (PDF/HWP → Markdown 변환 및 DB 저장) |
| 머신러닝 엔지니어 | [김명환](https://c0z0c.github.io) | 임베딩 처리 (Markdown → 벡터 임베딩 및 FAISS 저장)|
| 5      | 이민규 | LLM 기반 정보 추출 및 요약 시스템                      |
| 프론트엔드 엔지니어 | 오형주 | Streamlit UI 개발 및 통합                           |

### 2.2. 데이터베이스 설계

**핵심 DB: 2개 SQLite + FAISS**

- **documents.db**: TB_DOCUMENTS (문서 텍스트 저장)
- **chat_history.db**: chat_sessions, chat_messages (대화 히스토리)
- **FAISS Index**: vectorstore.faiss (벡터 + 메타데이터 통합)

> 📚 **상세 스키마**: [시스템 아키텍처 설계서](./시스템_아키텍처_설계서.md)

### 2.3. 핵심 처리 흐름

1. **문서 처리**: PDF → DocumentProcessor → documents.db
2. **임베딩**: EmbeddingProcessor → FAISS Index
3. **검색**: Retrieval (FAISS) → LLMProcessor
4. **UI**: Streamlit → 사용자 인터페이스

## 3. RAG 시스템 구현

### 3.1. 개발 우선순위

**더미 데이터 우선 개발 전략**

1. Step 1: 더미 데이터 생성
2. Step 2: DB 스키마 구축
3. Step 3: 모듈 구조 설계 (더미 함수)
4. Step 4: UI 프로토타입
5. Step 5: 기능 통합

### 3.2. 데이터 수집 및 전처리 (신승목)

**주요 작업**:
- PDF/HWP 파일 수집
- 파일 해시 (SHA-256) 계산
- PyMuPDF로 텍스트 추출
- documents.db에 저장

> 📚 **구현 세부사항**: [시스템 아키텍처 설계서](./시스템_아키텍처_설계서.md#모듈-구조)

### 3.3. 임베딩 및 벡터 DB (김명환)

**주요 작업**:
- LangChain RecursiveCharacterTextSplitter로 청킹
- OpenAI API로 임베딩 생성
- FAISS 인덱스 구축 (VectorStoreManager)
- Document.metadata에 파일명, 페이지 번호 저장

### 3.4. Retrieval & Generation (이민규)

**Retrieval**:
- FAISS 유사도 검색 (L2 distance)
- top_k 청크 반환

**Generation**:
- LangChain + OpenAI LLM
- 프롬프트 엔지니어링
- chat_history.db 저장

### 3.5. UI 구현 (오형주)

**Streamlit 앱**:
- 채팅 인터페이스
- 세션 관리
- 검색 결과 및 출처 표시
